# 개요

Stable의 USDT별 기능은 독립적인 옵션 메뉴가 아닙니다. 이들은 서로 구성됩니다. 각 기능은 스테이블코인 결제가 데모에서 프로덕션으로 전환될 때 발생하는 특정 마찰을 제거합니다. 이 페이지에서는 다섯 가지 기능이 함께 존재하는 이유를 설명합니다.

## 마찰 스택

범용 체인의 대부분의 스테이블코인 결제 아키텍처는 동일한 문제 스택에 직면합니다.

1. **사용자는 스테이블코인을 이동하는 트랜잭션의 가스 비용을 지불하기 위해 두 번째 토큰(ETH, SOL)을 보유해야 합니다.** 전환율을 저하시키는 온보딩 단계입니다.
2. **두 번째 토큰이 있더라도 사용자는 가스 비용을 부담해야 합니다.** 이는 상인과 결제 앱이 필요로 하는 "1달러에 1달러 송금"이라는 정신적 모델을 깨뜨립니다.
3. **네트워크 활동에 따라 거래 비용이 변동됩니다.** 급여, 재무 운영 및 일괄 결제는 비용 또는 포함을 계획할 수 없습니다.
4. **거래당 제한은 처리량을 제한합니다.** 대량 USDT 흐름은 체인 전체의 경합에 부딪히고 관련 없는 활동과 함께 저하됩니다.
5. **모든 거래는 공개적으로 관찰 가능합니다.** 공급자 지급, 급여 실행 및 재무 이동은 상업적으로 민감한 데이터를 유출합니다.

## 기능 구성 방법

Stable은 전용 메커니즘으로 각 마찰을 해결합니다.

| 마찰 | 메커니즘 | 페이지 |
| :--- | :--- | :--- |
| 별도의 가스 토큰 | USDT0는 기본 가스 토큰입니다 | [가스 토큰으로서의 USDT](/ko/explanation/usdt-as-gas-token) |
| 사용자가 가스를 전혀 지불하지 않음 | 거버넌스 승인 면제가 가스 비용을 부담합니다 | [가스 면제](/ko/explanation/gas-waiver) |
| 비용 및 포함 변동성 | 등록된 파트너를 위한 예약된 블록 용량 | [보장된 블록 공간](/ko/explanation/guaranteed-blockspace) |
| 처리량 상한선 | 병렬화된 USDT0 전송 일괄 처리 | [USDT 전송 Aggregator](/ko/explanation/usdt-transfer-aggregator) |
| 공용 금액 가시성 | ZK 암호화를 통한 선택적 프라이버시 | [기밀 전송](/ko/explanation/confidential-transfer) |

이들 중 하나라도 도움이 됩니다. 이들을 함께 사용하면 Stable은 결제 팀이 기존 카드 네트워크에서와 동일한 방식으로 비용, 지연 시간 및 개인 정보를 모델링할 수 있는 체인이 되며, 블록체인의 결제 완결성을 제공합니다.

범용 EVM 체인과 Stable의 전체 차이점에 대해서는 [이더리움 비교](/ko/explanation/ethereum-comparison)를 참조하십시오. 이러한 기능을 통해 USDT가 처음부터 끝까지 이동되는 작업 예시는 [자금 흐름](/ko/explanation/flow-of-funds)을 참조하십시오.
