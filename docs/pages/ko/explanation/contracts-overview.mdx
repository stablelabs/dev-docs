# Stable의 컨트랙트

Stable은 완전히 EVM 호환됩니다. Solidity, Vyper, Hardhat, Foundry, ethers.js, viem이 수정 없이 동작합니다. 도구를 Stable의 RPC로 가리키기만 하면 기존 컨트랙트가 그대로 배포됩니다. 표준 EVM 위에, Stable은 프로토콜 수준 모듈(Bank, Distribution, Staking)을 고정 주소의 프리컴파일된 컨트랙트로 노출하므로, Solidity 코드에서 이들을 다시 구현하지 않고도 스테이킹 및 보상 분배를 호출할 수 있습니다.

## 무엇을 만들 수 있나

- 모든 EVM 도구체인으로 만드는 **표준 애플리케이션 컨트랙트**(ERC-20, ERC-721, 에스크로, AMM).
- ethers.js를 통한 실시간 이벤트 스트림과 함께 Stablescan에서 **검증되고 인덱싱된 컨트랙트**.
- Solidity에서 Bank / Distribution / Staking 프리컴파일을 호출하는 **프로토콜 통합 컨트랙트**.
- 표준 `eth_getLogs`를 통해 프로토콜에서 발생한 이벤트(예: 언본딩 완료)를 감시하는 **시스템 트랜잭션 리스너**.

## Stable의 차이점

- **USDT0가 가스 토큰입니다.** `maxPriorityFeePerGas`는 `0`이어야 합니다. 네이티브 전송에서 `value` 필드는 ETH가 아니라 USDT0를 담습니다. [USDT0를 가스로 사용하기](/en/how-to/work-with-usdt-gas)를 참조하세요.
- **USDT0는 이중 역할을 합니다.** 네이티브 USDT0를 보유한 컨트랙트는 ERC-20 `transferFrom` 또는 `permit`에 의해 잔액이 변경될 수 있습니다 — 절대 네이티브 잔액을 `uint256`에 미러링하지 마세요. [Stable에서의 USDT0 동작](/en/explanation/usdt0-behavior)을 참조하세요.
- **프리컴파일 주소는 테스트넷과 메인넷에서 고정되어 있습니다.** 이를 상수로 컨트랙트에 박아 넣으세요.
