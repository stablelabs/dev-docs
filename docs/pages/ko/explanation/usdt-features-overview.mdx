# 개요

Stable의 USDT 특정 기능들은 독립적인 옵션 메뉴가 아닙니다. 이들은 서로 구성되어 있습니다. 각 기능은 스테이블코인 결제가 데모 단계를 넘어 프로덕션으로 전환될 때 나타나는 특정 마찰 요소를 제거합니다. 이 페이지에서는 이 다섯 가지 기능들이 왜 함께 존재해야 하는지 설명합니다.

## 마찰 스택

대부분의 범용 체인에서 스테이블코인 결제 아키텍처는 동일한 문제 스택에 직면합니다.

1. **사용자는 두 번째 토큰을 보유해야 합니다.** (ETH, SOL) 스테이블코인을 이동시키는 거래에 대한 가스비를 지불하기 위해서입니다. 온보딩 단계에서 전환율이 떨어집니다.
2. **두 번째 토큰이 있더라도 사용자가 가스비를 부담해야 합니다.** 이는 상인 및 결제 앱이 필요로 하는 "1달러를 보내면 1달러"라는 정신 모델을 깨뜨립니다.
3. **네트워크 활동에 따라 거래 비용이 변동됩니다.** 급여, 재무 운영, 배치 정산 등은 비용이나 포함 여부를 계획할 수 없습니다.
4. **거래당 제한은 처리량을 제한합니다.** 고량의 USDT 흐름은 체인 전체의 경쟁을 유발하고 관련 없는 활동과 함께 성능이 저하됩니다.
5. **모든 거래는 공개적으로 관찰 가능합니다.** 공급자 결제, 급여 지급, 재무 이동은 상업적으로 민감한 데이터를 누출합니다.

## 기능 구성 방식

Stable은 전용 메커니즘으로 각 마찰 요소를 해결합니다.

| 마찰 요소 | 메커니즘 | 페이지 |
| :--- | :--- | :--- |
| 별도의 가스 토큰 | USDT0은 기본 가스 토큰입니다 | [가스용 USDT0](/ko/explanation/usdt-as-gas-token) |
| 사용자가 가스비를 지불해야 함 | 거버넌스 승인 면제가 가스비를 충당합니다 | [가스 면제](/ko/explanation/gas-waiver) |
| 비용 및 포함 편차 | 등록된 파트너를 위한 예약된 블록 용량 | [보장된 블록 공간](/ko/explanation/guaranteed-blockspace) |
| 처리량 상한선 | 병렬 USDT0 전송 일괄 처리 | [USDT 전송 통합자](/ko/explanation/usdt-transfer-aggregator) |
| 공개 금액 가시성 | ZK 암호화를 통한 선택적 개인 정보 보호 | [기밀 전송](/ko/explanation/confidential-transfer) |

이들 중 어느 하나도 도움이 되지만, 함께 사용하면 Stable은 결제 팀이 전통적인 카드 네트워크에서와 동일한 방식으로 비용, 지연 시간 및 개인 정보 보호를 모델링할 수 있는 체인이 됩니다. 물론 블록체인의 최종 정산 기능도 함께 제공됩니다.

범용 EVM 체인과 Stable의 모든 차이점에 대해서는 [이더리움 비교](/ko/explanation/ethereum-comparison)를 참조하십시오. 이러한 기능을 통해 USDT가 어떻게 처음부터 끝까지 이동하는지에 대한 작동 예시는 [자금 흐름](/ko/explanation/flow-of-funds)을 참조하십시오.

## 다음 권장 사항

- [**가스용 USDT0**](/ko/explanation/usdt-as-gas-token): 이더리움을 대체하여 가스와 결제를 동시에 처리하는 자산을 이해합니다.
- [**자금 흐름**](/ko/explanation/flow-of-funds): 온램프에서 온체인 전송, 오프램프 정산까지 USDT를 추적합니다.
- [**USDT0를 Stable 블록체인으로 브리징하기**](/ko/explanation/usdt0-bridging): USDT0가 다른 체인에서 Stable로 이동하는 방법을 확인합니다.
