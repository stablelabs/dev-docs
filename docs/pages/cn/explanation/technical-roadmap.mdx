# 路线图

Stable 从三个阶段优化交易管道的每个层（共识、执行、存储、RPC 和 USDT 特定流程）。本页面标记已发布、正在进行和即将推出的功能。

## 阶段 1：USDT 的基础层

状态：**已在主网启动。**

### StableBFT：已上线

基于 CometBFT 构建的自定义 PoS 共识协议。提供确定性最终性，以及高达三分之一验证者数量的拜占庭容错能力。请参阅[共识](/cn/explanation/consensus)了解当前实现。

### USDT 作为原生 Gas：已上线

USDT0 是用于 Gas 支付和价值转移的原生资产，同时支持 ERC-20 接口 (`approve`、`transfer`、`transferFrom`、`permit`)。请参阅[USDT 作为 Gas 代币](/cn/explanation/usdt-as-gas-token)。

### Stable Pay 和 Stable Name：进行中

Stable Pay 是一种 Web2.5 UX 钱包体验，旨在简化新用户的入门，同时与现有 Web3 钱包兼容。Stable Name 是一种用户友好的别名系统，用人类可读的标识符替换原始 EVM 地址，用于发送和接收代币。

## 阶段 2：USDT 体验层

状态：**测试网上的 v1.4.0 候选版本。**主网升级计划待定。USDT 转移聚合器仍在开发中。

### MemIAVL 状态存储：包含在 v1.4.0 中

MemIAVL 用一个内存映射快照和仅追加的预写日志取代了 LevelDB。RocksDB 支持的 VersionDB 为早于保留快照的历史查询提供服务。请参阅[存储 (StableDB)](/cn/explanation/stable-db)。

### 乐观并行执行：包含在 v1.4.0 中

Block-STM 在多个 CPU 核心上并发执行事务，检测冲突并重新执行受影响的工作。固定的区块内顺序在验证者之间保留了确定性状态。请参阅[执行](/cn/explanation/execution)。

### 选择性 RecheckTx：包含在 v1.4.0 中

区块提交后，节点仅重新检查受影响账户的待处理事务。当应用程序不支持时，CometBFT 会回退到完整的内存池重新检查。

### 2D 随机数和保证的区块空间：包含在 v1.4.0 中

独立的随机数通道可防止一个卡住的事务阻塞同一账户的所有后续事务。协议保留的随机数密钥用于识别 VIP 流量，而每个通道的 Gas 配额则用于保留区块容量。请参阅[保证的区块空间](/cn/explanation/guaranteed-blockspace)。

### USDT 转移聚合器：计划中

USDT0 聚合机制，将 USDT0 转移分组并集体处理，从而降低每次事务的开销并提高整体吞吐量。请参阅 [USDT 转移聚合器](/cn/explanation/usdt-transfer-aggregator)。

## 阶段 3：USDT 的全栈优化层

状态：**计划中。**

### Autobahn 上的 StableBFT

基于 DAG 的 BFT 共识，与 Stable 基于 CometBFT 的共识层自然集成。请参阅[StableBFT](/cn/explanation/consensus)了解当前协议，并参阅[Autobahn](/cn/explanation/autobahn)了解目标架构。内部概念验证已在受控环境中展示了超过 200,000 TPS（仅共识）的性能。

### StableVM++

高性能执行引擎，用 C++ 实现取代基于 Go 的 EVM。预计可将 EVM 执行速度提高多达 6 倍。

### 高性能 RPC

一个完整的 RPC 堆栈，涵盖节点级增强（实时链状态处理）、节点集成索引（低延迟应用程序 API）、基于 WebSocket 的可伸缩 pub/sub 和一个按操作类型路由的混合负载均衡器。请参阅[高性能 RPC](/cn/explanation/high-performance-rpc) 了解当前的分割路径架构。
