# 终局性规则与兼容性保证

Stable 的交易在 EVM 执行环境中处理。当交易被打包入区块后，其状态变更立即生效。

### 确认规则：

- 交易被区块打包即视为确认
- 状态的变更（余额、存储、事件）可通过 RPC 查询

### 结算特性：

- Stable 实现单槽终局性（single-slot finality）：
- 交易一旦被有效区块打包即不可逆。
- 开发者可直接将区块打包视为结算完成。

### 兼容性：

- 系统模块接口与执行行为在测试网期间保持稳定
- 任何潜在的破坏性变更将提前公告并记录于变更日志
- 发布时附带迁移指引
- 后续版本将引入正式的兼容性政策与变更分类机制
