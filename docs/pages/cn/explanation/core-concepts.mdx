# 核心概念

四个概念足以开始构建。每个部分都定义了概念，展示了它，并链接到完整的参考资料。

## USDT0 作为 gas

您以 USDT0 支付交易费，这是您已经持有和交易的相同资产。没有第二种代币需要资金或管理。

USDT0 既是原生 gas 资产（18 位小数，通过 `address(x).balance` 读取），也是 ERC-20 代币（6 位小数，通过 `USDT0.balanceOf(x)` 读取）。这两个接口操作的是相同的底层余额，协议会自动协调 12 位精度的差距。

```solidity
// 两者都读取相同的余额：
uint256 native = address(user).balance;        // 18 位小数
uint256 erc20  = IERC20(USDT0).balanceOf(user); // 6 位小数
```

:::warning
余额对账会在备用地址 `0x6D11e1A6BdCC974ebE1cA73CC2c1Ea3fDE624370` 发出额外的 `Transfer` 事件。重放 `Transfer` 事件的索引器必须过滤进出该地址的转账，否则它们将无声地重复计算余额。
:::

阅读更多：[USDT0 作为 gas](/cn/explanation/usdt-as-gas-token) · [USDT0 在 Stable 上的行为](/cn/explanation/usdt0-behavior)。

## 保证区块空间

Stable 为预先分配的企业工作负载保留了每个区块容量的一部分。即使一般流量拥堵，保留流量也能以可预测的延迟和成本结算；它不参与费用市场竞争。

这种行为在调用者级别是透明的。您以正常方式提交交易；分配在协议级别适用于已注册账户。

阅读更多：[保证区块空间](/cn/explanation/guaranteed-blockspace)。

## USDT 传输聚合器

大容量 USDT0 传输通过 MapReduce 启发式管道并行批处理和验证。每个账户的失败都是隔离的，因此一个不良传输不会中止批处理。

调用者侧的传输 API 保持不变。您以正常方式提交传输，无需更改代码即可提高吞吐量。

阅读更多：[USDT 传输聚合器](/cn/explanation/usdt-transfer-aggregator)。

## EVM 兼容性

标准 EVM 工具可以不变地工作。在 EVM 层面，有三种行为与以太坊不同（上面已经介绍了 USDT0 作为 gas，这是第四种）。

**单槽最终性。** 交易一旦包含在区块中即为最终。区块大约每 0.7 秒产生一次。

**无优先级小费。**`maxPriorityFeePerGas` 始终被忽略。有效 gas 价格由协议设定的基础费用决定。

```typescript
import { ethers } from "ethers";

const block = await provider.getBlock("latest");
const baseFee = block.baseFeePerGas;

const tx = await wallet.sendTransaction({
  to: "0xRecipientAddress",
  value: ethers.parseEther("0.1"),
  maxFeePerGas: baseFee * 2n,        // 2 倍基础费用作为安全边际
  maxPriorityFeePerGas: 0n,          // 在 Stable 上始终为 0
});

await tx.wait();
console.log("Included at gas price:", tx.gasPrice?.toString());
```

```text
Included at gas price: 1000000000
```

**双重角色 USDT0，移植风险。** 从以太坊移植的合约不应镜像原生余额，应拒绝 `address(0)` 传输，并且不应依赖 `EXTCODEHASH` 进行地址重用检测。

:::warning
在 Stable 上，移植在内部变量中镜像原生余额的合约是不安全的。外部 `USDT0.transferFrom` 调用可以在不调用任何合约代码的情况下耗尽合约的原生余额。始终在传输Moment with `address(this).balance` 进行偿付能力检查。
:::

## 机密传输（计划中）

Stable 有一个针对零知识传输的计划功能，该功能可以在授权方可审计的情况下隐藏金额。它尚未上线。
