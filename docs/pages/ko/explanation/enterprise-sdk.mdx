# Stable Enterprise SDK

`@stablechain/enterprise`는 Stable의 Enterprise 트랜잭션 레일을 위한 서버 측 TypeScript 클라이언트입니다. 가스 면제 트랜잭션(사용자의 가스 비용을 후원하는 경우)과 블록 공간 보장 트랜잭션(예약된 Enterprise 레인에 포함되는 경우)의 두 가지 유형의 특권 트랜잭션을 서명하고 릴레이합니다. 하나의 클라이언트에서 두 레일 중 하나 또는 둘 다를 활성화할 수 있습니다.

```ts
import { createStableEnterprise, stableTestnet } from "@stablechain/enterprise";
import { privateKeyToAccount } from "viem/accounts";

const enterprise = createStableEnterprise({
  chain: stableTestnet,
  gasWaiver: { account: privateKeyToAccount("0xYOUR_WAIVER_KEY") },
});

const { txHash } = await enterprise.gasWaiver.send(user, { to: token, data });
```

```text
txHash: 0x8f3a...2d41
```

호출은 사용자의 트랜잭션 해시인 `H_inner`를 반환하며, 이를 포함하는 래퍼 트랜잭션은 반환하지 않습니다.

## SDK의 기능

SDK는 모듈당 세 가지 기능을 제공합니다. 각 기능은 선택 사항이며 독립적으로 구성되며, 자체 서명 계정을 가지고 있어 별도의 키를 사용할 수 있습니다.

- **가스 면제를 통한 릴레이** (`gasWaiver`): 가스 면제 트랜잭션을 구축, 서명 및 릴레이합니다. 화이트리스트에 등록된 면제 계정이 사용자의 무가스 트랜잭션을 래핑하여 사용자가 USDT0를 보유하지 않고도 트랜잭션을 처리할 수 있도록 합니다. [가스 면제](/ko/explanation/gas-waiver)를 참조하십시오.
- **블록 공간 보장** (`guaranteedBlock`): Enterprise RPC 게이트웨이를 통해 트랜잭션을 릴레이하여 예약된 Enterprise-레인 블록 공간에 포함되도록 합니다. 가스 면제와 달리 서명자는 자체 가스 비용을 지불합니다. [블록 공간 보장](/ko/explanation/guaranteed-blockspace)을 참조하십시오.
- **보장된 릴레이 트랜잭션** (`guaranteedWaiver`): 두 기능을 결합합니다. 가스 면제 트랜잭션이 보장된 블록 공간을 통해 라우팅되므로 사용자는 잔액이 필요 없으며 트랜잭션은 여전히 Enterprise 레인에 포함됩니다.

모든 기능은 동일한 네 가지 메서드를 제공합니다. SDK가 서명하는 관리형 경로를 위한 `send` 및 `sendBatch`, 그리고 사용자가 다른 곳에서 서명하고 서명된 헥스만 전달하는 비관리형 경로를 위한 `relay` 및 `relayBatch`입니다.

SDK는 npm에 [`@stablechain/enterprise`](https://www.npmjs.com/package/@stablechain/enterprise)로 게시되어 있으며, 피어 종속성으로 `viem >= 2.0.0`이 필요합니다.

## 접근 권한

Enterprise SDK는 현재 Stable Testnet에서만 사용할 수 있습니다.

두 레일 모두 게이트화되어 있습니다. 가스 면제에는 거버넌스에 등록된 면제 키가 필요하며, 블록 공간 보장에는 Enterprise RPC 게이트웨이 API 키가 필요합니다. 통합하려면 Stable에 [문의](https://discord.gg/stablexyz)하여 액세스 권한을 얻으십시오. Stable은 필요에 따라 면제 키와 게이트웨이 엔드포인트를 제공합니다.

## 서버 측 전용

:::warning
Enterprise SDK는 프라이빗 키로 서명하는 상태 비저장 서버 측 라이브러리입니다. 브라우저에서 실행하지 마십시오. 면제 키와 Enterprise RPC 게이트웨이 URL을 백엔드에 보관하십시오.
:::
